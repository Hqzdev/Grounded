# Code of Conduct

Grounded contributors are expected to keep collaboration direct, professional, and evidence-based.

## Standards

Expected behavior:

1. Critique ideas and code, not people.
2. Explain technical objections clearly.
3. Keep reviews actionable.
4. Respect security disclosure boundaries.
5. Keep discussions focused on project outcomes.

Unacceptable behavior:

1. Harassment.
2. Threats.
3. Public disclosure of private vulnerabilities.
4. Personal attacks in review.
5. Deliberate sabotage of project infrastructure or data.

## Enforcement

Maintainers may remove comments, reject contributions, revoke access, or block contributors who violate this policy.

## Review Culture

Grounded values direct review. A rejected change is not a personal failure. Weak architecture, missing tests, unsafe tenant behavior, and unclear ownership should be challenged early.
